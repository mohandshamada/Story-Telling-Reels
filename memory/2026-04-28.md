## What Happened
Started building the Story-to-Reels app from scratch. Went from an empty repo (only roadmap.md) to a fully scaffolded full-stack application.

## What Was Built

## Key Decisions
- Used **MoviePy 2.x** (latest) — required rewriting video_compiler.py because API changed significantly from 1.x (`.resized()`, `.with_duration()`, `.with_effects([FadeIn(...)])`, etc.)
- SQLite for Phase 1 MVP (keeps dev simple, easy to swap to PostgreSQL later)
- ElevenLabs `eleven_multilingual_v2` model for TTS
- DALL-E 3 `1024x1792` size (closest to 9:16)
- Local file storage mounted as `/storage` static files
- Celery worker runs single-threaded (`-c 1`) to avoid event loop issues with async services

## Second Build Session (Same Day)

### What Was Built

#### 1. PostgreSQL Migration
- Added `psycopg2-binary` to dependencies
- `db/session.py` now auto-detects SQLite vs PostgreSQL from `DATABASE_URL`
- `docker-compose.yml` now includes `postgres:16-alpine` service
- SQLite remains the default for local dev (zero-config), PostgreSQL for Docker Compose

#### 2. Arabic Full Support
- `app/services/arabic_utils.py` — complete pipeline:
  - `normalize()` — removes tatweel (kashida), normalizes alef variants, standardizes hamza
  - `add_diacritics()` — lightweight heuristic tashkeel (adds fatha to consonants without diacritics)
  - `reshape_for_display()` — uses `arabic-reshaper` + `python-bidi` for PIL rendering
  - `estimate_duration()` — Arabic spoken at ~3.5 chars/sec (denser than English)
- `story_parser.py` — automatically preprocesses Arabic narration before TTS
- `video_compiler.py` — all text rendering (subtitles, title cards, outro) now reshapes Arabic for correct RTL display

#### 3. Karaoke Word-by-Word Subtitles
- Replaced simple TextClip subtitles with **PIL-based frame generator**
- Each word gets timed proportionally to its character length
- Active word is:
  - Highlighted in **yellow** (`255, 223, 100`)
  - Rendered at **115% font size**
  - Has a subtle text shadow for readability
- Inactive words are white with black stroke shadow
- Works seamlessly with Arabic (RTL) thanks to `reshape_for_display`
- Word wrapping handles multi-line subtitles automatically
